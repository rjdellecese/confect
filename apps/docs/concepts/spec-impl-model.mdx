Your Confect API is broken up into two parts: a **spec** and its implementation (or **impl**).

## Spec

A spec defines the interface of your Confect API. It is made up of group specs and function specs. Each function spec defines the function's name, arguments schema, and returns schema -- but not the function's logic.

Specs are built using `@confect/core`, which means they can be shared between the server and client. This separation is what enables end-to-end schema decoding and encoding: the client knows the exact shape of every function's arguments and return value without importing any server code.

Leaf specs live in colocated `*.spec.ts` files and **default-export** their `GroupSpec`. Group names are derived from the filesystem path at codegen time — authors use `GroupSpec.make()` without a name:

```ts confect/notesAndRandom/notes.spec.ts
import { GroupSpec, FunctionSpec } from "@confect/core";
import { Schema } from "effect";

import { Notes } from "../tables/Notes";

export default GroupSpec.make().addFunction(
  FunctionSpec.publicQuery({
    name: "list",
    args: Schema.Struct({}),
    returns: Schema.Array(Notes.Doc),
  }),
);
```

The CLI assembles all leaf specs into `confect/_generated/spec.ts`. Do not hand-write a root `spec.ts`.

## Impl

An impl provides the logic for each function declared in your spec. Each leaf impl default-imports its sibling spec (conventionally bound to a variable named after the group — the `*.spec.ts` stem) and passes that group object to `FunctionImpl.make` and `GroupImpl.make` — no dot-path strings:

```ts confect/notesAndRandom/notes.impl.ts
import { FunctionImpl, GroupImpl } from "@confect/server";
import { Effect, Layer } from "effect";

import api from "../_generated/api";
import { DatabaseReader } from "../_generated/services";
import notes from "./notes.spec";

const list = FunctionImpl.make(api, notes, "list", () =>
  Effect.gen(function* () {
    const reader = yield* DatabaseReader;

    return yield* reader
      .table("notes")
      .index("by_creation_time", "desc")
      .collect();
  }).pipe(Effect.orDie),
);

export default GroupImpl.make(api, notes).pipe(Layer.provide(list));
```

Each Convex module imports a per-group registry from `confect/_generated/registeredFunctions/{path}` that pulls in **only** that leaf impl. There is no root `impl.ts`.

Run `confect codegen` after changing specs or impls.
