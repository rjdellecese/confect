`@confect/react` provides drop-in replacements for Convex's React hooks. Each hook automatically encodes your args and decodes return values through the [Effect Schemas](/concepts/schema-restrictions) defined in your [function specs](/concepts/spec-impl-model). You work with the Schema `Type` (decoded) values on both sides тАФ the hooks handle the round-trip to Convex's `Encoded` representation transparently.

Functions are referenced via `refs` (from `confect/_generated/refs`) instead of Convex's `api` object. Each ref carries the `args`, `returns`, and (optional) `error` schemas from the corresponding function spec, which is what enables the automatic encoding, decoding, and [typed error](/server/typed-errors) recovery.

## Setup

The React provider setup is the same as vanilla Convex.

```tsx src/main.tsx
import { ConvexProvider, ConvexReactClient } from "convex/react";
import React from "react";
import ReactDOM from "react-dom/client";

import App from "./App";

const convexClient = new ConvexReactClient(
  import.meta.env.VITE_CONVEX_URL,
);

ReactDOM.createRoot(
  document.getElementById("root") as HTMLElement,
).render(
  <React.StrictMode>
    <ConvexProvider client={convexClient}>
      <App />
    </ConvexProvider>
  </React.StrictMode>,
);
```

## `useQuery`

Encodes args using the spec's `args` schema, passes them to Convex, and decodes the result using the spec's `returns` schema. Returns a `QueryResult<A, E>` тАФ a tagged union with `Loading`, `Success`, and `Failure` variants тАФ instead of `A | undefined`.

Given this spec:

```ts confect/notes.spec.ts
import { FunctionSpec, GroupSpec } from "@confect/core";
import { Schema } from "effect";

import { Notes } from "./tables/Notes";

export const notes = GroupSpec.make("notes").addFunction(
  FunctionSpec.publicQuery({
    name: "list",
    args: Schema.Struct({}),
    returns: Schema.Array(Notes.Doc),
  }),
);
```

The hook accepts `{}` (the `Type` of `Schema.Struct({})`) as args and returns `QueryResult<readonly Notes.Doc["Type"][], never>`. Use `QueryResult.match` to render each variant:

```tsx
import { QueryResult, useQuery } from "@confect/react";
import refs from "../confect/_generated/refs";

const NoteList = () => {
  const notes = useQuery(refs.public.notes.list, {});

  return QueryResult.match(notes, {
    onLoading: () => <p>LoadingтАж</p>,
    onSuccess: (notes) => (
      <ul>
        {notes.map((note) => (
          <li key={note._id}>{note.text}</li>
        ))}
      </ul>
    ),
  });
};
```

### Query results

`QueryResult<A, E>` is a tagged union with three variants:

| Variant   | Description                                                                                                                                                                                               |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Loading` | The query has no result yet. Carries a `skipped: boolean` flag (exposed as the argument to `onLoading`) that is `true` when `"skip"` was passed as args, and `false` when a fetch is genuinely in flight. |
| `Success` | The query succeeded. Carries the decoded `returns` value.                                                                                                                                                 |
| `Failure` | The query failed with a [typed error](/server/typed-errors). Only present when the ref declares an `error` schema. Carries the decoded error.                                                             |

`QueryResult.match` requires `onFailure` only when the ref declares an `error` schema тАФ for refs without typed errors, `E` is `never` and the failure variant is unreachable. You can also use `QueryResult.isLoading`, `QueryResult.isSuccess`, and `QueryResult.isFailure` directly if you prefer predicates over `match`.

Failures that fall outside the ref's `error` schema (network errors, decode errors, contract violations) are thrown from the hook, so they propagate to the nearest React error boundary rather than producing a `Failure` variant.

### Skipping queries

Pass `"skip"` instead of args to disable the query subscription. The hook returns a `Loading` variant with `skipped: true`. This is useful for conditional queries where a prerequisite value may not yet be available тАФ use the `skipped` flag inside `onLoading` to distinguish "actively fetching" from "idle, waiting for input".

```tsx
import { QueryResult, useQuery } from "@confect/react";
import refs from "../confect/_generated/refs";

const NoteDetail = ({
  selectedId,
}: {
  selectedId: string | undefined;
}) => {
  const note = useQuery(
    refs.public.notes.get,
    selectedId !== undefined ? { id: selectedId } : "skip",
  );

  return QueryResult.match(note, {
    onLoading: (skipped) =>
      skipped ? <p>Select a note</p> : <p>LoadingтАж</p>,
    onSuccess: (note) => <p>{note.text}</p>,
  });
};
```

### Typed errors

When a ref declares an `error` schema, `QueryResult` gains a `Failure` variant carrying the decoded error, and `onFailure` becomes required in `QueryResult.match`. See [Typed Errors](/server/typed-errors) for the full pattern.

```tsx
const lookup = useQuery(refs.public.notes.getOrFail, { noteId });

QueryResult.match(lookup, {
  onLoading: () => "Looking upтАж",
  onSuccess: (note) => `Found: ${note.text}`,
  onFailure: (error) => `Note ${error.noteId} not found.`,
});
```

## `useMutation`

Returns a function that encodes args using the spec's `args` schema, calls the Convex mutation, and decodes the result using the spec's `returns` schema.

The return shape depends on whether the ref declares an `error` schema:

- **No `error` schema:** the returned function resolves to `Promise<Returns>`, matching `convex/react`'s behavior.
- **With an `error` schema:** the returned function resolves to `Promise<Either<Returns, Error>>`. The success value is on the `Right`, the decoded typed error on the `Left`. Match on the `Either` to handle each branch. See [Typed Errors](/server/typed-errors) for details.

Given this spec:

```ts confect/notes.spec.ts
FunctionSpec.publicMutation({
  name: "insert",
  args: Schema.Struct({ text: Schema.String }),
  returns: GenericId.GenericId("notes"),
});
```

The returned function accepts `{ text: string }` and returns a `Promise<GenericId<"notes">>`:

```tsx
import { useMutation } from "@confect/react";
import refs from "../confect/_generated/refs";

const InsertNote = () => {
  const insertNote = useMutation(refs.public.notes.insert);

  return (
    <button onClick={() => void insertNote({ text: "Hello" })}>
      Insert note
    </button>
  );
};
```

Failures that fall outside the ref's `error` schema reject the returned promise тАФ wrap the call in `try`/`catch` or chain `.catch(...)` to handle them.

## `useAction`

Same pattern as `useMutation`: encodes args via the spec's `args` schema, calls the Convex action, and decodes the result via the spec's `returns` schema. Return shape follows the same rule тАФ `Promise<Returns>` without an `error` schema, `Promise<Either<Returns, Error>>` with one.

Given this spec:

```ts confect/random.spec.ts
FunctionSpec.publicAction({
  name: "getNumber",
  args: Schema.Struct({}),
  returns: Schema.Number,
});
```

The returned function accepts `{}` and returns a `Promise<number>`:

```tsx
import { useAction } from "@confect/react";
import refs from "../confect/_generated/refs";

const RandomNumber = () => {
  const getRandom = useAction(refs.public.random.getNumber);

  const handleClick = () => {
    void getRandom({}).then(console.log);
  };

  return <button onClick={handleClick}>Get random</button>;
};
```

## Differences from vanilla Convex hooks

| Vanilla Convex                                            | Confect                                                                                                          |
| --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| Functions referenced via `api.module.fn`                  | Functions referenced via `refs`                                                                                  |
| Args passed directly to Convex as-is                      | Args are schema-encoded from `Type` to `Encoded` before sending to Convex                                        |
| Return values received directly from Convex as-is         | Return values are schema-decoded from `Encoded` to `Type` before returning                                       |
| `useQuery` returns `Returns \| undefined`                 | `useQuery` returns `QueryResult<Returns, Error>` тАФ a tagged `Loading` / `Success` / `Failure` union              |
| `useMutation` / `useAction` resolve to `Promise<Returns>` | Same when no `error` schema is declared; `Promise<Either<Returns, Error>>` when one is                           |
| No typed-error support тАФ every failure rejects/throws     | Failures matching the ref's `error` schema surface as decoded values; everything else rejects/throws as before   |
| Pass `"skip"` to disable query subscription               | Same тАФ pass `"skip"` to disable query subscription. Reified as `Loading.skipped === true` on the result.         |
