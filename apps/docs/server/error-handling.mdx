Confect functions can declare an optional `error` schema. When a handler fails with a value matching that schema, Confect encodes the error, transports it as a Convex [`ConvexError`](https://docs.convex.dev/functions/error-handling/application-errors#throwing-application-errors), and surfaces the decoded value as a typed error at every call site — [`useQuery`](/clients/react), [`useMutation`](/clients/react), [`useAction`](/clients/react), [`HttpClient`](/clients/js/http), [`WebSocketClient`](/clients/js/websocket), and [`TestConfect`](/guides/testing).

Functions without an `error` schema pay nothing for the feature, and the encoded payload sits in `ConvexError.data` — the `returns` channel stays clean and non-Confect callers of the same API see native Convex semantics.

## Declaring a typed error

Pass an `error` schema to any `FunctionSpec` constructor. Use a `Schema.TaggedError` for a single failure mode, or `Schema.Union` to combine multiple shapes.

```ts confect/notes.spec.ts
import { FunctionSpec, GenericId, GroupSpec } from "@confect/core";
import { Schema } from "effect";

import { Notes } from "./tables/Notes";

export class NoteNotFound extends Schema.TaggedError<NoteNotFound>()(
  "NoteNotFound",
  { noteId: GenericId.GenericId("notes") },
) {}

export const notes = GroupSpec.make("notes").addFunction(
  FunctionSpec.publicQuery({
    name: "getOrFail",
    args: Schema.Struct({ noteId: GenericId.GenericId("notes") }),
    returns: Notes.Doc,
    error: NoteNotFound,
  }),
);
```

## Failing with a typed error

The matching [function impl](/server/functions) can `Effect.fail` (or `Effect.mapError` to) any value the declared schema accepts. Confect encodes it through that schema and wraps it in a `ConvexError` at the function boundary.

```ts confect/notes.impl.ts
import { FunctionImpl } from "@confect/server";
import { Effect } from "effect";

import api from "./_generated/api";
import { DatabaseReader } from "./_generated/services";
import { NoteNotFound } from "./notes.spec";

const getOrFail = FunctionImpl.make(
  api,
  "notes",
  "getOrFail",
  ({ noteId }) =>
    Effect.gen(function* () {
      const reader = yield* DatabaseReader;

      return yield* reader
        .table("notes")
        .get(noteId)
        .pipe(Effect.mapError(() => new NoteNotFound({ noteId })));
    }),
);
```

Failures that do not match the declared schema — including handlers that never declare an `error` schema — surface as defects on the client and continue to reject the underlying `Promise` or fail the `Stream` with the original error.

## Consuming typed errors

### Effect call sites

[`HttpClient`](/clients/js/http), [`WebSocketClient`](/clients/js/websocket), and [`TestConfect`](/guides/testing) surface the decoded error in the `Effect` error channel alongside the existing transport and decoding errors.

```ts
import { HttpClient } from "@confect/js";

HttpClient.query(refs.public.notes.getOrFail, { noteId });
// Effect.Effect<Note, NoteNotFound | HttpClientError | ParseError>
```

Recover from the typed error with `Effect.catchTag` (for `Schema.TaggedError`s), `Effect.catchTags`, or `Effect.catchAll`:

```ts
Effect.gen(function* () {
  const client = yield* HttpClient.HttpClient;

  return yield* client.query(refs.public.notes.getOrFail, { noteId });
}).pipe(
  Effect.catchTag("NoteNotFound", ({ noteId }) =>
    Effect.succeed(`No note with id ${noteId}`),
  ),
);
```

### React hooks

[`@confect/react`](/clients/react) hooks expose typed errors through dedicated result shapes. See the React clients page for `QueryResult.match` examples and the `Either` return shape used by `useMutation` and `useAction` when a ref declares an `error` schema.
