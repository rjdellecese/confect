Confect's HTTP API integration lets you define HTTP endpoints using Effect 4's HTTP API modules (`effect/unstable/httpapi` and `effect/unstable/http`) and mount them onto Convex's HTTP router. It includes built-in interactive API documentation powered by [Scalar](https://github.com/scalar/scalar).

## Defining endpoints

Define your API groups and endpoints using `effect/unstable/httpapi`.

```ts confect/http/notes.ts
import {
  HttpApi,
  HttpApiBuilder,
  HttpApiEndpoint,
  HttpApiGroup,
  OpenApi,
} from "effect/unstable/httpapi";
import { Effect, Layer, Schema } from "effect";

import refs from "../_generated/refs";
import { QueryRunner } from "../_generated/services";
import { Notes } from "../tables/Notes";

class NotesGroup extends HttpApiGroup.make("notes")
  .add(
    HttpApiEndpoint.get("getFirst", "/get-first", {
      success: Schema.Option(Notes.Doc),
    }).annotate(
      OpenApi.Description,
      "Get the first note, if there is one.",
    ),
  )
  .annotate(OpenApi.Title, "Notes")
  .annotate(OpenApi.Description, "Operations on notes.") {}

export class Api extends HttpApi.make("Api")
  .annotate(OpenApi.Title, "My App")
  .add(NotesGroup)
  .prefix("/api") {}
```

<Note>
  `HttpApiEndpoint.get(name, path)` accepts an options object with
  `success`, `error`, `params`, `query`, `payload`, and `headers`
  fields.
</Note>

## Implementing handlers

Implement your endpoint handlers in the same file using `HttpApiBuilder.group`. All Confect action [services](/concepts/services) (`QueryRunner`, `MutationRunner`, `ActionRunner`, `Scheduler`, `Auth`, `ActionMeta`, `StorageReader`, `StorageWriter`, `StorageActionWriter`) are available in handlers.

```ts confect/http/notes.ts (continued)
const NotesGroupLive = HttpApiBuilder.group(
  Api,
  "notes",
  (handlers) =>
    handlers.handle("getFirst", () =>
      Effect.gen(function* () {
        const runQuery = yield* QueryRunner;

        return yield* runQuery(refs.public.notes.getFirst, {});
      }).pipe(Effect.orDie),
    ),
);

export const ApiLive = HttpApiBuilder.layer(Api).pipe(
  Layer.provide(NotesGroupLive),
);
```

<Note>
  Each group is provided as its own `Layer` (built with
  `HttpApiBuilder.group(Api, name, build)`) and composed onto
  `HttpApiBuilder.layer(Api)` via `Layer.provide`.
</Note>

## Creating the router

Create the Convex HTTP router in `confect/http.ts` using `HttpApi.make` from `@confect/server`. Each key is a path prefix that must be either `"/"` or end with a trailing slash (e.g. `"/api/"`).

```ts confect/http.ts
import { HttpApi } from "@confect/server";
import { HttpMiddleware } from "effect/unstable/http";
import { flow } from "effect";

import { Api, ApiLive } from "./http/notes";

export default HttpApi.make({
  "/api/": {
    api: Api,
    apiLive: ApiLive,
    middleware: flow(HttpMiddleware.cors(), HttpMiddleware.logger),
  },
});
```

Each path prefix entry accepts:

| Property | Description |
| --- | --- |
| `api` | The `HttpApi` value used for route reflection and Scalar docs |
| `apiLive` | A `Layer` providing the HTTP API, built with `HttpApiBuilder.layer` |
| `middleware` | Optional middleware function (e.g. `HttpMiddleware.cors()`, `HttpMiddleware.logger`) |
| `scalar` | Optional [Scalar configuration](https://github.com/scalar/scalar) to customize the interactive API docs |

## API documentation

Interactive API docs are automatically served at `{pathPrefix}docs` (e.g. `/api/docs`). The docs are powered by Scalar and generated from your endpoint definitions and OpenApi annotations.
