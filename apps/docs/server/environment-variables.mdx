Confect provides a `ConfigProvider` that is compatible with the Convex runtime. This provider is automatically set for all Confect functions (queries, mutations, actions, and HTTP API handlers), so Effect's [`Config`](https://effect.website/docs/configuration/config/) module works out of the box.

## Reading environment variables

Use `Config` from Effect to read [Convex environment variables](https://docs.convex.dev/production/environment-variables) in your function handlers.

```ts confect/settings.impl.ts
import { FunctionImpl } from "@confect/server";
import { Config, Effect } from "effect";

import api from "./_generated/api";

const getApiKey = FunctionImpl.make(
  api,
  "settings",
  "getApiKey",
  () => Config.string("API_KEY").pipe(Effect.orDie),
);
```

Common Effect `Config` combinators work as expected—`Config.string`, `Config.number`, `Config.boolean`, `Config.withDefault`, `Config.option`, etc.

```ts
Config.integer("MAX_RESULTS").pipe(Config.withDefault(100));
```

## Typed Convex env definitions

Convex 1.39 added typed env declarations in `defineApp` and `defineComponent`. Confect can derive key-safe `Config` helpers from those declarations:

```ts convex/convex.config.ts
import { defineApp } from "convex/server";
import { v } from "convex/values";

const app = defineApp({
  env: {
    API_KEY: v.string(),
    MODE: v.union(v.literal("dev"), v.literal("prod")),
  },
});

export default app;
```

```ts confect/settings.impl.ts
import { ConvexConfigProvider, FunctionImpl } from "@confect/server";
import { Effect } from "effect";

import type app from "../convex/convex.config";
import api from "./_generated/api";

const env = ConvexConfigProvider.fromApp<typeof app>();

const getMode = FunctionImpl.make(api, "settings", "getMode", () =>
  env.string("MODE").pipe(Effect.orDie),
);
```

For components, use `ConvexConfigProvider.fromComponent<typeof component>()`.

If your Confect files and `convex.config.ts` live in separate TypeScript projects, use a local env shape instead:

```ts
const env = ConvexConfigProvider.fromEnv<{
  API_KEY: string;
  OPTIONAL_API_KEY?: string;
}>();
```

For optional declarations, use the key-safe `option` helper. It wraps Effect's `Config.option(Config.string(key))` and returns an `Option.Option<string>`.

```ts
const optionalApiKey = env.option("OPTIONAL_API_KEY");
```

## Custom config provider

The `ConvexConfigProvider` module is exported from `@confect/server` if you need to create a provider with a custom env snapshot. It supports the same options as Effect's default [`ConfigProvider.fromEnv`](https://effect.website/docs/configuration/config-provider/#fromenv).

```ts
import { ConvexConfigProvider } from "@confect/server";

const provider = ConvexConfigProvider.make({
  env: {
    API_KEY: "secret",
  },
});
```

<Note>
  Internally, `ConvexConfigProvider.make()` snapshots `process.env` at
  construction time and forwards it to `ConfigProvider.fromEnv(...)`.
  Effect 4's default `ConfigProvider.fromEnv` would read
  `import.meta.env`, which the Convex bundler cannot statically
  analyze; passing the env explicitly side-steps that bundler
  limitation.
</Note>

To use a custom provider directly, parse the config with that provider.

```ts
const getApiKey = FunctionImpl.make(
  api,
  "settings",
  "getApiKey",
  () => Config.string("API_KEY").parse(provider).pipe(Effect.orDie),
);
```
